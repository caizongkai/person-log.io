# 配置maven项目

1、 新建任务，选择maven项目

2、 “源码管理”Tab中设置git信息和分支，选择刚才在全局凭证中添加的GitLab账户密码

3、 “Build”Tab设置pom文件路径，和maven执行命令，构建开发环境我们这里选择打包application-dev.yml配置文件

maven命令参考：

clean install package -Dmaven.test.skip=true -P dev
1

2、 “构建后操作”Tab添加SSH，把在Jenkins服务器上打包好的jar部署到远程服务器上

“Exec command”是指在远程服务器上执行的命令，我们在73服务器上编写好脚本，所以这里只需要执行脚本即可。

start_app.sh脚本内容如下，作为参考：

#!/bin/bash
if [ ! -n "$1" ]; then
        echo "项目名为空，终止操作"
else
echo "检查$1"
source_file="/data/app/temp/$1.jar"
file_dir="/data/app/$1/"
file_name="$1.jar"
file_log="/data/logs/$1.log"
file_path=${file_dir}${file_name}

if [ -f "${source_file}" ]; then
        echo "移动$1.jar到目标文件夹"
        mv ${source_file} ${file_dir}
        echo "${file_name} 文件已更新"
else
echo "临时目录不存在该文件：${source_file}，尝试重启应用"
fi

if [ -f ${file_path} ]; then
echo "准备重启应用"
#下面写停止和重新部署代码 根据服务名强制杀死进程
kill -9 `ps -ef | grep ${file_name} | grep -v "grep" | awk '{print $2}'`
sleep 3s
cd ${file_dir}
nohup java -jar ${file_name} > ${file_log} &

else
echo "应用目录不存在该文件：${file_path}，终止操作！"
fi
fi
